# 法律关系图可视化编辑 Skill

## 适用场景

- 法律关系图
- 案件事实图
- 交易结构图
- 诉讼路径图
- 争议焦点图

## 标准工作流

1. 先用文本资料提炼主体、合同关系、争议事实、案件路径。
2. 生成初版 `nodes` / `edges` / `sections` / `timelineItems` / `legendItems` 数据。
3. 进入编辑模式，检查主体位置、分区层次、标签位置和连线路径。
4. 由用户在浏览器中人工拖拽修正节点、紫色关系标签和线条。
5. 保存 JSON，形成可复用、可回滚的数据版本。
6. 导出 SVG / PNG，用于汇报材料、案件说明、PPT 或邮件正文。

## 数据维护规则

- 主体 / 权利主张主体使用蓝色：`subject`
- 货物 / 库房 / 标的物使用黄色：`object`
- 争议事实 / 风险事件使用红色：`dispute`
- 诉讼或程序路径使用绿色：`case`
- 合同 / 协议 / 程序动作标签使用紫色 `edge.label`
- 实线表示已发生事实或合同关系：`dashed: false`
- 虚线表示争议主张或待确认关系：`dashed: true`
- 文字优先使用中文短句，复杂事实放到节点正文或附表，不要把全部细节塞进连线标签。
- 坐标细节优先由用户在可视化界面调整，Codex 主要维护结构化数据和功能逻辑。

## 后续给 Codex 的标准提示词模板

### 新建一张法律关系图

请基于当前 React + TypeScript + SVG 法律关系图编辑器，新建一套图数据。不要重构组件逻辑，只根据以下案件材料生成 `nodes`、`edges`、`sections`、`timelineItems`、`legendItems` 初版数据。主体用 `subject`，标的物用 `object`，争议事实用 `dispute`，案件路径用 `case`，合同和程序动作放在 `edge.label`。

### 只修改 JSON 数据

请只修改导出的 `legal-diagram-data.json`，不要改 React 组件逻辑。根据以下修改意见调整节点文字、坐标、尺寸、关系标签和连线路径，保持现有数据结构。

### 不改组件逻辑，只改 nodes / edges

请只修改 `defaultDiagramData` 中的 `nodes` 和 `edges`，不要改导出功能、拖拽功能、属性面板、localStorage 或页面结构。调整目标是让图更适合汇报阅读。

### 根据案件材料生成初版图

请阅读以下案件材料，提炼：
1. 交易主体；
2. 合同关系；
3. 货物流和货权主张；
4. 核心争议事实；
5. 案件路径和时间顺序。

然后生成可导入编辑器的 JSON 数据，文件名为 `legal-diagram-data.json`。

## 注意事项

- 不要让 Codex 反复微调坐标。
- 坐标、标签细节和线条避让优先由用户在可视化界面拖拽调整。
- Codex 主要负责结构化数据、初版布局、功能维护、导入导出和构建修复。
- 复杂连线可以先通过属性面板编辑 `path` 实现，不必一开始做复杂的可视化折线控制点。
- 每次对外发送前，建议保存 JSON，再导出 SVG / PNG。
